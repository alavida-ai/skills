example-skill

linear-pipeline

A two-phase workflow that transforms raw input into structured output — first thinking about what to build, then building it with quality gates at every transition.

When This Skill Fires
User Request
Explicit ask for the skill's output — a structured analysis, transformation, or deliverable.
Proactive
The agent detects a pattern that would benefit from this skill — complex input that needs structure.
Pipeline Step
Another skill or orchestrator invokes this skill as part of a larger workflow.
triggers the two-phase workflow
Phase 1 — Analyze

Understand before you transform

Read the source material, classify its structure, and build a processing plan. Every decision downstream depends on getting the classification right — simple inputs get direct transformation, complex inputs get iterative refinement.

Phase 2 — Transform

Structured output with quality gates

Apply the processing plan to produce structured output. The user reviews the result before delivery. A final quality gate catches issues that automated checks miss — rejected output loops back for refinement.
